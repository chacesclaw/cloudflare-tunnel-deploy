# 🔑 如何获取 Cloudflare Global API Key（避免搞错的完整指南）

> **重要提示：** 本项目的部署脚本**只需要 Global API Key**，不是 API Token。
>
> 这两个东西**长得不一样**，位置也不一样，很多人会搞混——读完这篇就不会错了。

---

## 🧩 首先：搞清楚 API Key 和 API Token 的区别

| 项目 | Global API Key ⬅️ 本项目的需要 | API Token |
|------|-------------------------------|-----------|
| **格式** | 以 `cfk_` 开头 | 以 `cfut_`、`cft_` 或自定义开头 |
| **长度** | 37 位字符 | 40~60 位字符 |
| **权限** | 拥有账户**全部权限**（等同于密码） | 可自定义权限范围（精细化控制） |
| **创建方式** | 自动生成，**不能自定义** | 手动创建，可选模板 |
| **本项目的用途** | ✅ 需要（创建 Tunnel、DNS 记录等） | ❌ 本项目不用 |

>   **最容易搞错的地方**：Cloudflare 的 API Tokens 页面有两个区域——
> - 上半部分是 **API Tokens**（`cfut_` / `cft_` 开头）
> - 下半部分是 **Global API Key**（`cfk_` 开头）
> 
> 你可能已经注意到了，我们两个之前给你的都是 `cfut_` 开头的 API Token，不是 Global API Key！

---

## 📱 完整操作步骤（图文指引）

### 第一步：登录 Cloudflare

打开 https://dash.cloudflare.com/ ，登录你的账号。

### 第二步：打开 API Tokens 页面

点击右上角的用户头像 → **"我的个人资料"**（My Profile）

![用户头像 → 我的个人资料]

然后在左侧菜单中，点击 **"API 令牌"**（API Tokens）

![左侧菜单 → API 令牌]

### 第三步：找到 Global API Key

进入 API Tokens 页面后，**往下滚动**，不要被页面前半部分的 "API Tokens" 迷惑。

**关键：Global API Key 区域在页面的下半部分！**

```
 ┌─────────────────────────────────────────┐
 │  API Tokens（这里是上半部分）             │
 │  ┌──────────────────────────────────┐   │
 │  │  cfut_xxxxxxxxxxxxxxxxxxxxxxxxxx │   │ ← 这是 API Token，不是我们要的
 │  └──────────────────────────────────┘   │
 │  ...                                    │
 │  ...（继续往下滚）...                     │
 │  ...                                    │
 │                                          │
 │  API Keys（这里是下半部分）               │
 │  ┌──────────────────────────────────┐   │
 │  │  Global API Key: cfk_xxxxxxx    │   │ ← 我们要的是这个！
 │  │  [ View ]  [ Copy ]             │   │
 │  └──────────────────────────────────┘   │
 └─────────────────────────────────────────┘
```

**看到了吗？它有明显的 "Global API Key" 标签。**

### 第四步：查看并复制

1. 点击 **"View"** 按钮（或 "查看"）
2. Cloudflare 会要求你**再次输入密码**进行安全验证
   - ⚠️ 这是正常的安全措施，不是被骗了
3. 输入密码后，Global API Key 会显示出来
4. 点击 **"Copy"** 按钮复制它

**请把它保存在安全的地方**，它的作用和你的账户密码一样重要！

## 🖼️ 一个更不容易搞错的方法

如果你觉得以上步骤还是容易混淆，可以用这个**最笨但最安全**的方法：

1. 打开 https://dash.cloudflare.com/profile/api-tokens
2. 页面会分成两个板块：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  API Tokens（不要这里）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  这里列出的是你创建的 API Token
  格式：cfut_开头、或者你自定义的名字
  创建按钮文字： "创建令牌"（Create Token）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  API Keys（要这里！）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  Global API Key
  格式：cfk_开头
  操作按钮： "查看"（View）→ "复制"（Copy）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

记住一句话：**找到 "Global API Key" 这几个字，它下面的就是你要的东西。**

---

## ⚠️ 安全提示

| 可以 ✅ | 不可以 ❌ |
|---------|----------|
| 在本项目的部署脚本中使用 | 把 Key 公开到 GitHub、博客、社交平台 |
| 保存在本地密码管理器 | 通过不加密的渠道传输 |
| 随时在 Cloudflare 后台重新生成 | 告诉你不信任的人 |

**如果怀疑 Key 泄露：** 立刻回到 [API Tokens](https://dash.cloudflare.com/profile/api-tokens) 页面，点击 Global API Key 区域的 **"重新生成"（Regenerate）**，旧的 Key 会立即失效。

---

## ✅ 快速核对清单

- [ ] 你复制的是以 `cfk_` 开头的字符串（不是 `cfut_` 或 `cft_`）
- [ ] 字符串长度大约为 37 个字符
- [ ] 它位于 API Tokens 页面的 **下半部分 "API Keys"** 区域
- [ ] 标签明确写着 **"Global API Key"**
- [ ] ✅ 这就是部署脚本需要的东西，可以继续下一步了！
